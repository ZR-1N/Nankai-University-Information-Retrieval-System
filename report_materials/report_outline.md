# 实验报告大纲

## 基于南开大学多源网站资源的校内信息检索系统

> 100000条文档规模，22项功能验证通过，45个测试通过

---

### 1. 项目背景与目标
- 信息检索技术概述
- 高校校内信息检索需求
- 项目目标：多源采集、全文检索、Web服务

### 2. 数据源配置
- **v2.0**: 8个南开大学网站（主站、新闻网、计算机学院、AI学院、教务部、研究生招生网、校史网、导师网）
- **v3.0**: 扩展至30个来源（seed_urls.json），运行时自动发现至36个活跃源
  - 理学部：数学、物理、化学、生科、环境、统计
  - 医学部：医学、药学
  - 经管学部：经济、商学、金融、旅服
  - 工学/信息：软件、网安、电光、材料
  - 人文社科：文学、历史、哲学、外语、法学、周政
  - 直属/职能：图书馆、人事处、国际合作处、就业中心、国教、汉院
- seed_urls.json 配置结构（单源含多个种子URL，含category/page_role标注）
- WordPress类站点（校史网）添加?p=NNNN详情页模式支持

### 3. 系统架构
- 总体架构图（爬虫→索引→检索→Web）
- 模块划分：crawler/, indexer/, search_engine/, web/, database/, tests/
- 技术选型：Python + Flask + Jinja2 + Bootstrap 5 + SQLite

### 4. 多源均衡爬取
- Round-Robin轮询算法（36个活跃source_site队列）
- **v3.0优化**: 单源配额从35%降至**30%**，当活跃源>3时自动启用
- **v3.0新增**: 动态配额重分配 — 队列耗尽时配额自动分配给活跃源
- **v3.0新增**: source_depleted追踪 — 耗尽标记防止空源占位
- **30000→100000条效果分析**：
  - 30000条阶段：36源均匀分布，最大source占比仅6.1%，均衡策略效果显著
  - 100000条阶段：部分学院队列渐次耗尽，南开新闻网作为最大公共新闻源占比41.4%
  - 系统仍保留36个南开相关来源，100000条有效资源规模
- BFS广度优先 + 4级优先级队列（document > detail > list > portal）
- URL规范化 + 双重去重（URL hash + content hash）
- 断点续爬 + checkpoint机制
- 请求频率控制（1.0s 延迟）
- 100000条爬取耗时约20.4小时，平均速度1.36页/秒，失败URL 1708，重复页面6073

### 5. 页面分类与内容提取
- 页面角色分类：portal/list/detail/document
- 最终分布：detail 80958 (81.0%), list 13879 (13.9%), document 5147 (5.1%), portal 16
- URL模式匹配 + 内容启发式判断
- **v3.1 保守修正**: 标题为已知栏目名（"通知公告""新闻中心"等）自动识别为list页
- 多策略标题提取（og:title → CSS选择器 → h1 → title → h2 → anchor_text）
- 导航噪声检测与去除（nav_noise仅40条，占比约0.0%）
- 正文提取与质量评估（good/short/nav_noise/failed）
- **关键修复**：移除form标签的误删除（AI学院页面正文位于form内）

### 6. 文档附件处理
- 附件URL发现（扩展名 + 链接文本关键词）
- 格式支持：PDF/DOC/DOCX/XLS/XLSX/ZIP/RAR/PPT（共8种格式）
- 文件解析（PyMuPDF+pdfplumber回退 / python-docx / openpyxl含warning抑制）
- **v3.0 关键修复**：MuPDF C层stderr泄露彻底解决 — os.dup2()在文件描述符层面重定向fd=2到日志文件
- parse_status字段：ok 98023 (98.0%), failed 1623 (1.6%), skipped 351 (0.4%)
- 5147个文档附件（PDF 1449 + DOC 1208 + DOCX 1262 + XLSX 556 + XLS 268 + ZIP/RAR 351 + PPT 53）

### 7. 倒排索引
- jieba中文分词 + 停用词过滤
- 标题倒排索引 + 正文倒排索引
- Posting格式与持久化
- 100000条 → 32983标题词条 + 367979正文词条（构建时间240.5秒）

### 8. BM25排序
- BM25算法（k1=1.5, b=0.75）
- 多字段加权（标题3.0, 正文1.0）
- 时间新鲜度加成（7/30/90/365天）
- 来源权威度加成
- nav_noise降权（-5.0），failed降权（×0.3）

### 9. 查询功能
- 普通查询（OR模式）
- 精确查询（exact boost）
- 多关键词查询（AND/OR自动推断）
- 文档查询（file_type筛选）
- 高亮摘要生成
- 匹配解释说明

### 10. 通配查询
- `*` → `.*`（多字符匹配）
- `?` → `.`（单字符匹配）
- 正则转换与词表匹配
- 通配无匹配不回退全量文档
- 100000条规模：`奖学*`→2744条，`研究??`→6061条

### 11. 模糊查询
- Levenshtein编辑距离（≤2）
- 动态规划实现
- 查询词扩展与模糊匹配分数
- 100000条规模演示：
  - `研宄生`→2059条（"宄"→"究"形近字容错）
  - `奖学今`→346条（"今"→"金"错字容错）
  - `奖学斤`→86条（"斤"→"金"同音字容错）

### 12. 个性化排序
- 5个用户画像（默认/本科生/研究生/教师研究人员/行政人员）
- 规则化关键词加权
- 来源偏好加权

### 13. 搜索联想

### 14. Web前端

- 响应式设计 + 搜索联想下拉

### 15. 查询日志
