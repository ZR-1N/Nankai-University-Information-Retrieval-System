## 五、模糊查询（Levenshtein 编辑距离 ≤ 2）

演示：容错匹配、编辑距离计算、模糊词扩展

| # | 查询 | 结果数 | 扩展词示例 | 推荐原因 |
|---|------|--------|-----------|---------|
| 1 | `奖学今` | 346 | 奖学金, 奖助学金 | **主推演示** — 经典错字演示（今 → 金），结果精确 |
| 2 | `奖学斤` | 86 | 奖学金, 奖助学金 | 同音错字"斤"→"金"，结果更精确 |
| 4 | `培养芳案` | 12684 | 培养方案, 培养计划 | 错字"芳"→"方"，展示编辑距离匹配 |
| 5 | `申请标` | 10341 | 申请表, 申请书, 申请材料 | 同音错字"标"→"表" |

### 验证命令

```bash
python run.py search "奖学今" --type fuzzy
python run.py search "奖学斤" --type fuzzy
python run.py search "培养芳案" --type fuzzy
python run.py search "申请标" --type fuzzy
```

### 模糊查询说明

- `奖学今` / `奖学斤` → `奖学金`：利用编辑距离≤2容忍输入错误（今→金、斤→金）
- `培养芳案` → `培养方案`：错别字"芳"→"方"被自动纠正
- **注意**："复式"→"复试"在索引中返回0条，因为"复试"的词频特征与"复式"差异过大，不建议用于演示
- **注意**："研宄生"→"研究生"虽然理论上可以匹配，但实际 Top 结果偏向"交换生"等单字"生"的匹配，演示效果不稳定，不作为推荐演示词

## 八、索引中不存在日期的说明

当前索引中的日期格式为 `YYYY-MM-DD`（如 2026-06-14），jieba 分词按字符分割，`2026-06-14` 作为一个整体 token 存入索引，而 `2026年06月14日` 会被拆分为 `2026年`+`06月`+`14日`。

因此 `20??年`、`202?-06` 等日期通配符在当前索引中返回 0 —— 这些模式的 token 不存在于索引词典中。

**如果需要日期搜索，建议使用普通查询模式搜索年份数字（如 `2026`），而非通配符。**
